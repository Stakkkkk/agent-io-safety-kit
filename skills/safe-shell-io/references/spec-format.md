# Формат command spec

JSON-файл должен быть UTF-8 без BOM. Не хранить в нём секреты дольше выполнения задачи.

Для редакторской подсказки можно добавить:

```json
{
  "$schema": "../../../schemas/command-spec.schema.json"
}
```

Каноническая схема находится в `schemas/command-spec.schema.json` в корне комплекта.

## Поля

- `command` — обязательная непустая строка с программой или исполняемым файлом.
- `args` — массив строк; каждый элемент передаётся как отдельный argv без shell-разбора.
- `cwd` — рабочий каталог. Относительный путь разрешается от каталога spec.
- `env` — объект дополнительных переменных окружения. Строка задаёт значение, `null` удаляет переменную из окружения дочернего процесса.
- `stdin` — строка, передаваемая процессу как UTF-8.
- `stdinFile` — путь к файлу с байтами stdin. Нельзя использовать одновременно с `stdin`.
- `stdoutEncoding`, `stderrEncoding` — `utf8`, другое имя кодировки `TextDecoder` или `raw`. По умолчанию `utf8` со строгой проверкой.
- `stdoutFile`, `stderrFile` — записать нормализованный поток в файл вместо терминала. Относительный путь разрешается от `cwd`.
- `timeoutMs` — положительный таймаут. По умолчанию 30000 мс.
- `maxOutputBytes` — максимум каждого выходного потока. По умолчанию 16 MiB.

## Семантика

- Runner всегда использует `shell: false`.
- Текстовые потоки строго декодируются из указанной кодировки и повторно выводятся как UTF-8 без BOM.
- Режим `raw` не декодирует поток. Использовать его для бинарных данных или только вместе с файловым выводом.
- Код возврата runner совпадает с кодом дочернего процесса. Таймаут возвращает 124.
- `.cmd` и shell built-ins не являются переносимыми executable. Выбирать реальный бинарный файл или отдельный скрипт вместо включения shell.
