# Политика локализации

[English version](../localization.md)

## Канонический язык

Английский является каноническим для:

- основной GitHub/npm-витрины;
- основных файлов проекта;
- правил и skills по умолчанию;
- snippets для `--lang en`;
- issue templates и CI metadata.

Так проект остаётся понятным для более широкой open-source аудитории и даёт агентам технический английский, который они обычно обрабатывают стабильнее.

## Поддерживаемая русская локализация

Русский поддерживается как полноценная локализация, а не как второстепенное примечание.

Локализованные файлы используют `.ru.md` или лежат в `docs/ru/` и `snippets/ru/`. Установщик может развернуть русские инструкции:

```sh
node scripts/deploy.mjs --target /path/to/project --entry AGENTS.md --lang ru
```

Целевые пути остаются каноническими (`RULE.md`, `skills/.../SKILL.md`), поэтому агентам не нужно знать language-specific имена файлов.

## Правило для contribution

Если меняются канонические инструкции, по возможности обновляйте русскую локализацию в том же pull request. Если это невозможно, явно укажите, какая локализация осталась не обновлена.

Запустите `npm run check:localization` для проверки обязательных пар, уровней headings, code fences и canonical skill frontmatter. `npm run check:skills` проверяет имена skills и ограничения frontmatter без внешних Python/YAML dependencies.
